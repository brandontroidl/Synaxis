Synaxis Coding Style
────────────────────

Based on original X3/srvx conventions:

- K&R style: return type on separate line above function name
- 4-space indentation, no tabs in new code
- Braces on same line for if/for/while
- Function declarations: static before return type
- Use MODCMD_FUNC(name) for command handlers
- Use HasOps(mn) instead of (mn->modes & MODE_CHANOP)
- All OpenSSL access through proper abstractions
- No AI attribution in source files
